# 開発ルール

## ブランチ運用

- `main`: 完成版・提出用
- `dev`: 開発統合ブランチ
- `feature/backend-xxx`: バックエンド作業
- `feature/frontend-xxx`: フロントエンド作業
- `pm/xxx`: PM・仕様・ドキュメント作業
- `qa/xxx`: QA・テスト作業

## 基本の作業手順

1. `dev` から作業ブランチを切る
2. 自分の担当Issueに沿って作業する
3. 作業ブランチをpushする
4. `dev` 向けにPull Requestを出す
5. レビュー後に `dev` へマージする

## 禁止事項

- `main` に直接pushしない
- `dev` に直接pushしない
- 1つのPRで関係ない変更をまとめない
- MVP外の機能を勝手に増やさない

## PRで必ず書くこと

- やったこと
- 関連Issue
- 確認したこと
- 気になる点

## MVPの優先順位

1. ルームを作成できる
2. URLから参加できる
3. 商品名・価格・数量を入力できる
4. 誰が何を追加したか見える
5. 採用・却下できる
6. 合計金額と商品別数量が見える
