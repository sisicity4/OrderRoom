## 概要

イベントや買い出しで、複数人の希望・注文・購入候補を1つのルームにまとめられるWebアプリ。

代表者がLINEを見返したり、
「○○、何が欲しいんだっけ？」
と何度も確認したりしなくても、
全員の希望、採用された買い物リスト、合計金額をその場で確認できる。
# コンセプト

「注文をまとめる」のではなく、

**"みんなで一つの注文票を作る"**

という考え方のアプリ。

イベントの買い出しや飲み会準備では、
希望・予算・購入候補がLINEや口頭に散らばりやすい。

しかし、学生同士や友達同士で利用すると、

- 欲しいものをLINEで送る
- 誰が何を希望したか分からなくなる
- 合計金額を手計算する
- 代表者だけが忙しい

といった問題が起きる。

オーダールームは、
希望や注文を一人に集めるのではなく、
**全員で同じ注文票を編集する**ことで、イベント準備を楽にする。

---

# 設計の核

注文の主体は「人」ではなく「ルーム」。

全員が同じ注文票を見ながら、
それぞれが自分の注文だけを編集する。

代表者は最後に内容を確認し、
採用するものを決めて買い物や注文に進む。

```
ルーム
 ├── 太郎
 │      ├── ポテト
 │      └── コーラ
 │
 ├── 花子
 │      └── ハンバーガー
 │
 └── 次郎
        ├── ナゲット
        └── シェイク
```

この構造にすることで、

- 個人別注文
- 個人別金額
- 商品ごとの個数
- 全体金額

をすべて同時に計算できる。
