# OrderRoom ドキュメント案内

更新日: 2026-07-18

OrderRoomは、大学生のイベントで、欲しい物の提案から採否、買い物、実購入額、均等割り、立替精算記録までを一つのルームで扱うWebアプリである。

## 最初に読む資料

1. [ロードマップ](%E3%83%AD%E3%83%BC%E3%83%89%E3%83%9E%E3%83%83%E3%83%97.md) — 期限、優先順位、Slice、完成判定
2. [MVP仕様](MVP.md) — 作るもの、作らないもの、権限、完成シナリオ
3. 自分の担当資料 — 画面、API、QA、PMの詳細

## 仕様が矛盾した場合

1. ロードマップの期限・MVP範囲・完成条件を優先する
2. 次にPMの確定記録を確認する
3. 詳細は担当する専門資料を使う
4. 矛盾を見つけた資料だけを黙って直さず、影響する資料も確認する

## 用語

| 使用する | 使用しない |
| --- | --- |
| 提案 | 注文登録 |
| 見送り | 却下 |
| proposed | pending |
| 見積単価 | 価格だけの曖昧な表記 |
| 採用済み見積 | 全体合計 |
| 実購入額 | 購入後価格 |
| 負担額・立替額・精算額 | 個人別合計 |
