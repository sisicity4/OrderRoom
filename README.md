 MVP

## 概要

複数人の希望や注文を**ひとつのルームで集約し、誰が何をどれだけ欲しいか**を確認できるサービス。

---

## MVPで実現すること（スコープ）

### 1. ルーム作成（代表者）

- 注文ルームを新規作成できる
- 参加用のURL（必要ならQRコード）を発行・共有できる

### 2. ルーム参加（参加者）

- URL（またはQR）からルームに参加できる
- **名前のみ**入力して注文を開始できる（ログイン不要）

### 3. 注文登録（参加者）

- 以下を入力して注文を追加できる
    - 商品名
    - 価格
    - 数量
- 各注文に「追加した人（参加者名）」が紐づいて保持される

### 4. 自動集計

注文が追加・更新されたあと、画面表示時または更新操作時に以下を自動で計算・表示する（集計値は保存しない）。
リアルタイム同期はMVPでは必須にしない。

- 個人別の合計金額
- 全体の合計金額
- 商品ごとの合計数量

---

## データ設計（最小）

### 保存するデータ

- ルーム
- ユーザー（表示名のみでも可）
- 注文（商品名 / 価格 / 数量 / 追加者 / 作成日時 など）

### 計算で出すデータ（保存しない）

- 個人合計
- 全体合計
- 商品別数量

---

## 画面（最小）

- ルーム作成
- 参加（名前入力）
- 注文入力
- 注文一覧
- 集計（一覧と同画面でも可）

---

## 完成条件（MVPの成功基準）

- 代表者がルームを作成できる
- 参加者全員がルームに参加し、注文を入力できる
- 代表者が1画面で次を確認できる
    - 誰が何を注文したか
    - 全体で何を注文するか（商品別数量）
    - 合計金額

## バックエンドの起動方法

### 前提

- Java 21
- Neon（PostgreSQL）の接続情報（`DB_URL` / `DB_USER` / `DB_PASSWORD`）

### 1. 環境変数の設定

`backend/orderroom-backend/` で、Neonの接続情報を環境変数として設定します。

```bash
export DB_URL="jdbc:postgresql://<Neonのホスト>/<DB名>?sslmode=require"
export DB_USER="<ユーザー名>"
export DB_PASSWORD='<パスワード>'
```

### 2. 起動

```bash
cd backend/orderroom-backend
./mvnw spring-boot:run
```

`Tomcat started on port 8080` と表示されれば起動成功です。

### 3. 動作確認

```bash
curl -i http://localhost:8080/api/health
```

`{"status":"OK"}` が返れば正常です。
